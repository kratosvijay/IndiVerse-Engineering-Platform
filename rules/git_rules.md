# Git & Contribution Rules

Standardized Git branch management and commit protocols.

## Branch Naming
- Features: `feature/sprint-<id>-<description>`
- Bugfixes: `bugfix/<description>`
- Releases: `release/v<version>`

## Commit Message Format
We enforce **Semantic Commits**:
- `feat: <description>` (new capability)
- `fix: <description>` (bug correction)
- `docs: <description>` (only documentation adjustments)
- `style: <description>` (linting or styling formatting)
- `test: <description>` (adding/improving tests)
- `chore: <description>` (dependency updates, configuration changes)
